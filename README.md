# Telegram Account Shop Bot

Б0т-магазин Telegram-аkkаунтов на `aiogram`, `Telethon` и SQLite.

## Возможности

- Каталог аkkаунтов по странам.
- Баланс пользователей и история покупок.
- Покупка akkаунта с выдачей телефона, 2FA и кода из служебного чата Telegram.
- Таймаут сделки: если покупатель не завершил вход за 10 минут, товар возвращается в каталог, а деньги возвращаются на баланс.
- Админ-панель: добавление аккаунтов, склад, статистика, управление балансами, страны каталога, прокси, экспорт базы.
- Пополнение баланса российской картой, украинской картой или через CryptoPay.
- Ручные заявки на пополнение с проверкой чека в отдельных чатах операторов.
- Добавление аккаунтов по номеру телефона или загрузкой `.session` файла.
- Массовая загрузка `.session` файлов.
- Проверка живости аккаунтов через Telethon.
- Опциональное пополнение через CryptoPay.

## Структура

```text
start.py                  # Точка входа
bot.py                    # Старый совместимый алиас
shopbot/app.py            # Основные handlers, меню, покупки, админка
shopbot/db.py             # SQLite-схема и операции с данными
shopbot/session_flow.py   # Telethon: логин, коды, проверка аккаунтов
shopbot/keyboards.py      # Inline-клавиатуры
shopbot/config.py         # Настройки из config.local.json и env
shopbot/proxy_utils.py    # SOCKS/HTTP/MTProto proxy helpers
shopbot/paths.py          # Пути runtime/data/session файлов
data/shop.db              # SQLite база, создается автоматически
data/sessions/            # Session-файлы аккаунтов
data/runtime/             # Временные runtime-файлы
```

## Установка

```bash
pip install -r requirements.txt
```

## Конфигурация

Создайте `config.local.json` в корне проекта:

```json
{
  "BOT_TOKEN": "123456:telegram-bot-token",
  "ADMIN_IDS": [123456789],
  "API_ID": 123456,
  "API_HASH": "telegram-api-hash",
  "CRYPTOPAY_TOKEN": "",
  "SHOP_TITLE": "Stryx Shop",
  "CURRENCY": "RUB",
  "REQUIRED_CHANNEL": "@stryxxs",
  "RUB_TO_UAH_RATE": 0.45
}
```

Обязательные поля: `BOT_TOKEN`, `ADMIN_IDS`, `API_ID`, `API_HASH`.

## Запуск

```bash
python start.py
```

При старте бот сам создает `data/shop.db` и нужные папки.

## Основной сценарий покупки

1. Пользователь открывает каталог и выбирает товар.
2. `purchase_product()` атомарно списывает баланс и переводит товар в `waiting_code`.
3. Покупатель получает телефон, 2FA и кнопку запроса кода.
4. Бот читает свежий код из чата Telegram `777000` через session-файл товара.
5. После подтверждения входа товар переводится в `sold`.
6. Если вход не завершен за 10 минут, товар возвращается в каталог, а баланс покупателя пополняется на сумму сделки.

## Админка

Доступна администраторам из `ADMIN_IDS`.

- `/admin` - открыть админ-панель.
- `/search` - поиск пользователя по Telegram ID.
- `/delbalance <user_id>` - обнулить баланс пользователя.

Через inline-меню доступны:

- добавление аккаунтов;
- управление складом;
- редактирование товара;
- скачивание `.session`;
- выдача баланса;
- управление странами каталога;
- общий прокси;
- скан аккаунтов;
- экспорт базы в Excel;
- сброс статистики.

## Данные

Основные таблицы:

- `users` - пользователи, баланс, принятие условий.
- `products` - товары и данные аккаунтов.
- `purchases` - история покупок.
- `balance_events` - события баланса.
- `catalog_countries` - кнопки стран каталога.
- `crypto_payments` - обработанные CryptoPay invoice ID для защиты от повторного начисления.
- `topup_requests` - ручные заявки на пополнение, чеки и статусы проверки.

## Заметки по эксплуатации

- Для SOCKS/HTTP прокси нужны зависимости из `requirements.txt`.
- Для `.session` файлов используется Telethon-формат.
- Логи пишутся в `bot.log`.
- `config.local.json`, база данных и session-файлы не стоит коммитить в репозиторий.
- `RUB_TO_UAH_RATE` используется как резервный курс, если внешний курс RUB -> UAH временно недоступен.
