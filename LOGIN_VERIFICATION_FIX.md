# Исправления логики проверки входа покупателя

## Выявленная проблема
Функция `detect_buyer_login` должна искать **НОВЫЕ входы** - те, которые произошли **после начала сделки** (после `sold_at`). Без правильного ограничения по времени, система не может отсеять старые входы из истории аккаунта и может работать неправильно.

## Решение
Переделана логика `detect_buyer_login` для поиска входов **только с момента `sold_at`** (начало сделки). Все входы, произошедшие после этого момента - это новые входы в контексте именно этой сделки.

## Что было изменено

### 1. shopbot/db.py
- Добавлена колонка `code_fetched_at` в таблицу `products` (миграция автоматическая)
- Добавлена функция `update_code_fetched_at(product_id)` для сохранения времени получения кода (используется для логирования и статистики)

### 2. shopbot/session_flow.py
- **fetch_code_from_telegram**: После успешного получения кода сохраняет время в БД (для логирования)
- **detect_buyer_login**: 
  - Упрощена логика: **ВСЕГДА ищет с момента `sold_at`** (начало сделки)
  - Игнорирует старые входы, которые могли быть раньше в истории аккаунта
  - Буферизация -30 сек от `sold_at` для учета задержек сервера Telegram
- **_looks_like_new_login_notice**: Расширены маркеры для более надежного детектирования сообщений о входе
- Добавлено детальное логирование для отладки

### 3. shopbot/app.py
- Упрощены оба вызова `detect_buyer_login` (убрана передача `code_fetched_at`)

## Логика работы

1. **Покупатель покупает товар** → `sold_at` записывается в БД (начало сделки)
2. **Получает код** → `code_fetched_at` сохраняется в БД (для логирования и статистики)
3. **Вводит код в Telegram и выполняет вход** → приходит служебное сообщение о входе от @Telegram
4. **Нажимает "✅ Я вошел в аккаунт"** → бот ищет сообщения о входе **с момента `sold_at`**
   - Находит ТОЛЬКО новые входы в контексте этой сделки
   - Игнорирует старые входы, которые были до покупки товара

## Как тестировать

1. Перезапустите бота обычным образом
2. Приложение автоматически добавит колонку `code_fetched_at` при инициализации БД
3. Купите товар (или используйте товар #22 из `waiting_code`)
4. Нажмите "Получить код"
   - Проверьте логи: должно быть `Время получения кода сохранено в БД`
5. Введите код в Telegram и выполните вход в аккаунт
6. Нажмите "✅ Я вошел в аккаунт"
   - Проверьте логи: должны быть логи `Ищу НОВЫЕ входы после начала сделки` с правильным временем
   - Если вход найден: `✅ НАЙДЕН ВХОД`
   - Товар должен автоматически перейти в статус `sold`

## Логирование для отладки

Смотрите логи с префиксом `[product=XX]`:
- **`Ищу НОВЫЕ входы после начала сделки`** - граница поиска установлена правильно (с `sold_at`)
- **`✅ НАЙДЕН ВХОД`** - сообщение о входе успешно найдено
- **`Вход не найден`** - сообщение не найдено (возможные причины: текст сообщения от Telegram не совпадает с маркерами, сессия неавторизована, сообщение не пришло)

## Возможные проблемы и решения

### Вход не находится даже после введения кода
- Проверьте логи на текст сообщения о входе (должны быть в логах как `msg {N}: ...`)
- Текст может не совпадать с маркерами в `_looks_like_new_login_notice`
- Убедитесь, что сессия товара авторизована (`await client.is_user_authorized()`)

### Старые входы мешают проверке
- Их больше не должно быть, так как ищем только с момента `sold_at`
- Если проблема все еще есть - проверьте, что `sold_at` записывается правильно
