## 💾 Хранилище данных

### Структура JSON-файлов

```
data/
├── accounts.json       # Все аккаунты с деталями
├── balances.json       # Текущие балансы пользователей
├── transactions.json   # История всех операций
├── countries.json      # Список доступных стран
├── feedback.json       # Сообщения от пользователей
└── users.json          # Информация о пользователях
```

### Пример аккаунта:
```json
{
  "id": 1,
  "country": "Узбекистан",
  "phone": "+998901234567",
  "code": "12345",
  "twofa": "recovery_code",
  "price": 500.0,
  "sold": false,
  "sold_to": null,
  "sold_at": null,
  "added_at": "2024-01-01T12:00:00"
}
```

## ✨ Новые функции (обновление)

### 📲 Отправка кодов после покупки

При покупке пользователь получает:
- ✅ Основной код подтверждения в приватном сообщении
- ✅ Дополнительные коды (если добавлены администратором)
- ✅ 2FA данные (если указаны)
- ✅ Автоматические уведомления о новых кодах

### 🔧 Управление кодами

- **`/addcode`** - Добавить новый код к аккаунту
  - Администратор может добавлять перехваченные коды
  - Коды сохраняются последние 20
  - Пользователь автоматически уведомляется

    {"code": "98765", "timestamp": "2024-05-01T...", "source": "telegram_bot"}
  ],

## ✅ Чек-лист использования

- [ ] Установить зависимости: `pip install -r requirements.txt`
- [ ] Получить токен бота у @BotFather
- [ ] Получить свой Telegram ID
- [ ] Отредактировать `BOT_TOKEN` и `ADMIN_IDS` в bot.py
- [ ] Запустить: `python bot.py`
- [ ] Написать `/start` боту
- [ ] Добавить страну: `/addc`
- [ ] Добавить аккаунт: `/add`
- [ ] Проверить баланс пользователей: `/list`
- [ ] Выдать деньги: `/mn`
